## 📋 Launcher Changelog v0.6.22

### ✨ New Features

#### Process Logs & Crash Reports
- **Per-Session Game Log Capture** - Each launch redirects child stdout/stderr into `logs/game/{session}/nrc-process.log` + `session.json` instead of tailing the global `latest.log`, so each Minecraft run has its own isolated log on disk
- **Plain-Text Log4j Console Layout** - Patch the downloaded MC `client-X.X.xml` log4j config, replacing the new `<XMLLayout/>` console appender with a `[%d{HH:mm:ss}] [%t/%level]: %msg{nolookups}%n` `PatternLayout` so captured stdout/stderr stays human-readable instead of one-event-per-XML-block. `latest.log`'s file appender is untouched
- **Cursor-Polling Log Viewer** - In-app log windows now poll the process log via `get_process_log_cursor`, replacing the old per-line `process-output` event stream — far less IPC chatter and the viewer reconnects cleanly across window reopens
- **Process Logs Debug Tab** - New `list_process_logs` command + Debug → Process Logs sub-tab lists every captured session for quick inspection
- **Cursor-Based Crash Upload** - `GlobalCrashReportModal` uploads the full per-session `nrc-process.log` straight from disk instead of stitching together UI-buffered lines

#### Profiles
- **Block-Texture Icon Picker** - New `IconPicker` modal with a curated grid of ~104 Minecraft block textures (served from `InventivetalentDev/minecraft-assets`) plus a personal custom-icon library (uploads + URLs, persisted in localStorage). Animated block textures (vertical sprite sheets) auto-detected via aspect ratio and cropped to the first frame; ≤64px textures render pixel-crisp
- **Icon Picker in Wizard & Profile Settings** - Create-profile wizard step 3 shows a clickable icon avatar pre-filled with a random block; `ProfileIcon` in profile settings opens the same picker instead of the bare file dialog. `UploadProfileIconPayload` gained an `iconUrl` field so a URL icon can flow straight through `upload_profile_images`
- **Copy Profile ID Row** - Discreet copy-profile-ID row in the General settings tab
- **MC Version Sort Options** - "Newest Version" / "Oldest Version" sort options in the Profiles tab, ordering by `game_version` with numeric-aware comparison and name as tiebreaker

#### CLI
- **`launch` / `temp` / `version` Subcommands** - New `tauri-plugin-cli` subcommands, dispatched on both cold start (`main()`/`setup`) and hot start (single-instance forwarded argv)
- **Quick-Play Flags** - `--world <name>` / `--server <address>` (mutually exclusive) wire through to MC's quick-play args
- **Local Mod Injection** - `--mods <jar1,jar2,…>` passes local jars via a per-loader mods-folder JVM arg (fabric/forge/neoforge)
- **Account Override** - `--account <uuid>` picks a Mojang account for the launch
- **In-Memory Temp Profiles** - `nrc-launcher temp --mc 1.21.8 --loader neoforge` builds a throwaway profile in memory; swept to trash on next launcher start

#### NRC Pack Assets
- **Client-Managed Asset Mode** - For `nrc-nightly`, `nrc-standalone`, `nrc-mini` the launcher hands regular asset management to the in-game client. Launcher skips its own download/copy/cleanup for non-`overrides/` entries and passes `-Dnrc.assets.dir` / `-Dnrc.assets.bucket` so the client takes over. `overrides/` entries still flow through the launcher since they land in the profile game dir

### 🐛 Bug Fixes

#### Logs
- **`nrc-process.log` Size Cap** - Previously unbounded — mod stdout spam grew a single session log to 137 GB. Now uses log4rs `FixedWindowRoller` (same engine as `launcher.log`) for native gzip rotation: 10 MB live, 5 backups (`.1.gz` … `.5.gz`). Startup recovery truncates pre-fix files exceeding 500 MB. Session count cap lowered 50 → 10 [#130](https://staging.prime.gg/issues/130)

#### Launch & API
- **Standard Versions Endpoint** - `get_standard_version_profiles` now uses `launcher/versions-v3` instead of the legacy `launcher/versions`
- **No Mod Sync on Vanilla** - Skip the mod sync step entirely for vanilla profiles — vanilla doesn't load anything from `mods/`, so the previous sync/clean was pointless work (and also avoided creating an empty `mods/` directory)
