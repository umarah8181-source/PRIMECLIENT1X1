## 📋 Launcher Changelog v0.6.21

### ✨ New Features

#### Pack Rollout System
- **First Rollout Test** - Initial rollout infrastructure for staged pack rollouts
- **Interactive Rollout Indicator** - Pulsing bolt icon over the launch button with a glassmorphic hover popover anchored above the icon, showing rollout percentage from Flagsmith config and a Discord report link
- **Rollout Opt-Out Toggle** - Click the indicator to toggle `pack_rollout_override` between auto and off; opted-out icon goes greyscale
- **`is_pack_rollout_active` / `is_pack_aliased`** - New backend commands for bucket-membership and alias checks
- **Device-Stable Flagsmith Identity** - Use persistent `analytics_user_id` (localStorage) as Flagsmith identity so multivariate flag distributions are sticky per device, independent of Mojang account switches
- **Mojang Identity as Traits** - Mojang UUID and username are now set as Flagsmith traits instead of identity, allowing trait-based segments
- **`rollout_pct` Field** - New field on `PackRolloutConfig` for UI display
- **`-Dprime.pack` Game Property** - Launcher passes the selected pack id to the game [#50](https://staging.prime.gg/issues/50)

#### Backend Permissions
- **`PrimeApi.get_user_permissions`** - Calls `GET /api/v1/core/permissions` with the active account's bearer token
- **Permission Cache** - Cached in rust state via `OnceCell+RwLock` with sorted node list and last-fetched timestamp
- **Tauri Commands** - `refresh_permissions`, `get_cached_permissions`, `has_permission(node)` with LuckPerms-style wildcard matching ported from clientside-2's `NrcPermissionCache`
- **Auto-Refresh** - Permissions refresh on app mount and on account switch
- **Permissions Debug Tab** - Settings → Debug gets a new Permissions sub-tab listing all cached nodes with refresh button
- **Flagsmith → Backend Permissions Migration** - Experimental-mode toggle and designer profile-settings tab now gate on backend permissions (`prime.launcher.experimental`, `prime.command.designer`) via the new `usePermission` hook, sharing the same source of truth as the mod-client

#### Tester Window
- **Tester Window** - New window with needs-testing queue and voting flow

#### Profiles & UI
- **Sidebar on the Right** - Moved sidebar to the right in V3 profile detail view

#### Header
- **Unique-Players (24h) Carousel** - Header now fades between the launcher version and the unique-players-in-the-last-24h count from `/core/stats/uniquePlayers24h` every 8s; pauses on hover. Player slide shows a green online dot + count with a tooltip explaining the metric. Player slide only renders once a valid response is parsed
- **`get_unique_players_24h_command`** - New Tauri command (no-auth GET, parsed via `api_utils::parse_response_with_logging`)

#### Analytics
- **Language Change Tracking** - Analytics events for language changes
- **Launcher Start Tracking** - Analytics events for launcher start
- **Prime Pack Launch Tracking** - Analytics events for Prime pack launches
- **Crash Event Tracking** - Analytics events for crashes

### 🐛 Bug Fixes

#### Rollout & Packs
- **Phase-2 Mod Download Uses Effective Pack ID** - Phase-1 mod resolver already honored rollout aliases, but phase-2 prime pack mod download (`installer.rs:578`) was still using the original `selected_prime_pack_id`, causing inconsistent mod sets in cache when an alias was active
- **NRC Settings Tab Pack Dropdown** - Settings tab now shows the correct pack in the dropdown

#### Mods
- **ModDetailPage Install** - Installing a mod via ModDetailPage now works properly
- **Maven Forge Sorting** - Fixed inconsistent maven forge version sorting

#### Log Window
- **No White Flash on Open** - Log windows are built hidden, painted dark via inline HTML background, then shown after the first frame so the styled content is visible immediately
